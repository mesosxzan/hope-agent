# Hope Agent 0.1.0 — First Public Release

> [简体中文](./v0.1.0.md) · **English**

> Release date: 2026-05-08
> [`CHANGELOG.md`](../../CHANGELOG.md#010---2026-05-08) for the full list.

The same conversation hands off seamlessly between your devices and chat apps, and gets better the more you use it — cross-session memory accumulates, idle time gets spent organizing what mattered, and completed tasks crystallize into reusable skills. One native installer, GUI templates for the major model providers baked in, paste an API key and you're chatting; it also runs as a background service on a NAS, home server, or cloud VM, staying reachable through your IM apps wherever it lives.

## Highlights

- **Three run modes, one core**: desktop GUI (default) / `hope-agent server` (HTTP/WS daemon, installable as a system service with autostart) / `hope-agent acp` (stdio, as an IDE agent backend)
- **Cross-endpoint handoff**: the same conversation hands off seamlessly between desktop, browser, and IM — `/handover` pushes the current desktop session into a specific IM chat, `/session <id>` takes it over from the IM side; live desktop turns also **stream-mirror to IM**, so the model is literally typing into Telegram / Feishu / Slack as it thinks
- **Background service · browser as the client**: `hope-agent server start` runs an HTTP/WS daemon with the full Web GUI embedded (rust-embed) — phones, tablets, browsers can hit `http://<server>:port` and get a complete React UI without installing a client; Bearer Token + three-tier SSRF policy keeps public exposure controllable
- **First-launch onboarding wizard**: 11 steps on desktop and CLI alike — language, OpenClaw one-click import, local-vs-remote mode, Provider, user profile, Agent personality preset, approvals, skills, server bind+API key, IM channels, summary (Web GUI URL + LAN IPs + QR). Drafts persist mid-flow and resume on next launch; remote mode short-circuits to the first three steps once you point at an existing server
- **39 Provider templates · 206 preset models**: Anthropic / OpenAI / Codex (ChatGPT OAuth) / Gemini / OpenRouter / DeepSeek / Kimi / Qwen / Doubao / GLM / xAI / Mistral / Cerebras / DeepInfra / Tencent Hunyuan / Ollama — everything wired up out of the box
- **One-click local model install**: **no account, no API key, no terminal** — pick a Qwen / Gemma size that fits your hardware budget, and the wizard handles Ollama install, model download, Provider registration, and switching. Same flow covers the local Embedding model; a background watchdog keeps the default models hot-loaded in the Ollama runtime
- **Multi API-key auto-rotation**: multiple keys per Provider seamlessly switch on rate limit / quota exhaustion / auth failure; only if all are exhausted does it fall through to the next model
- **12 IM channels**: Telegram / Discord / Slack / Feishu / Google Chat / LINE / QQ Bot / Signal / iMessage / IRC / WeChat / WhatsApp. Inbound images / voice / files land in the multimodal context automatically; tool approvals can be clicked right inside the chat window. Reply mode picks one of three (split / final / preview) for typewriter / compact / progressive-preview rendering; `/reason` toggles thinking-stream visibility per channel
- **Cross-session persistent memory**: SQLite + FTS5 full-text + vector search combined, three-tier scope (global / project / agent), joint budget allocation so tiers don't squeeze each other out
- **Active recall + reflective profile + offline "dreaming"**: each turn pulls relevant memories into the prompt based on what you just said; reflection extracts a user profile from past conversations; on idle it auto-curates the most valuable memories from the past N days into a markdown diary
- **Skills that grow**: after finishing a complex task, drafts a new skill for your review; supports `paths:` conditional activation, fork-into-subagent execution, and per-skill tool allowlists; compatible with the agentskills.io open standard
- **Project container**: group related sessions together with project-level memory / instructions / shared files; uploaded files are auto text-extracted and loaded on demand
- **Agent Team multi-agent collaboration**: preset team templates in Settings — one natural-language line and the model spins up a specialist team to work in parallel
- **Plan Mode + natural-language cron + cron results piped to IM**: say "write me a daily report at 8 a.m." and the result can land right back in Telegram / Feishu when it fires; during planning, select any plan snippet to leave inline comments and have the model revise collaboratively
- **Unified permission engine v2**: Default / Smart / YOLO three session modes — Smart uses an independent LLM judge to auto-allow high-confidence calls; Protected Paths / Dangerous Commands / Edit Commands are three separate lists, and Protected Paths still force an approval prompt even under YOLO
- **Five-tier progressive context compaction + crash recovery**: no matter how long the conversation runs, it never hard-cuts; tool_use / tool_result pairs are never split across the boundary; every round is incrementally persisted, so a crash / `kill -9` / app close resumes from the last completed round on restart. Combined with prompt caching, long-session API cost drops dramatically vs. naïve calls
- **Background long-running tasks**: slow shell / web search / AI image generation can be dispatched to the background, returning `job_id` immediately without blocking the chat; results auto-inject when done
- **Built-in toolbox**: remote-controlled browser (CDP) / Canvas / AI image generation (7 Providers) / web search (8-Provider failover) / bash (optional Docker sandbox) / file read-write / grep / MCP / URL preview
- **Dashboard + `/recap` deep review**: 4-axis visualization (cost / tokens / activity heatmap / health score); one-click 11-section AI report exportable as self-contained HTML
- **12 UI languages**: Simplified / Traditional Chinese, English, Japanese, Korean, Spanish, Portuguese, Russian, Arabic, Turkish, Vietnamese, Malay; dark theme with carefully tuned typography

## Download & Install

Installers are attached to this Release — pick your platform and double-click. No need to pre-install Node, Rust, or Python.

- **macOS**: if the first launch complains about an unsigned app, allow it via **System Settings → Privacy & Security**
- **Windows**: installers are **not yet code-signed**, so SmartScreen may block them — choose **More info → Run anyway**. ConPTY is unavailable on Windows 10 1809 and earlier, which triggers a PTY fallback
- **Linux**: for AppImage, `chmod +x` and run directly; for deb, `sudo apt install ./hope-agent_*.deb`

## First Launch

1. The desktop wizard runs 11 steps on first start (language → OpenClaw import → local-vs-remote mode → Provider template / Codex OAuth → user profile → Agent personality preset → approvals → skills → server bind+API key → IM channels → summary). Closing mid-flow auto-resumes the draft on next launch; existing configured users are skipped; remote mode short-circuits to the first three steps
2. The embedded HTTP server binds to `127.0.0.1:8420` by default (local-only). Step 7 of the wizard lets you change the bind / set an API key, or edit `server.bindAddress` in `~/.hope-agent/config.json` to `0.0.0.0:8420` to expose on your LAN (strongly recommended to also set `server.apiKey`)
3. `hope-agent server install` registers the daemon as a system service with autostart — launchd on macOS, systemd user unit on Linux, Task Scheduler on Windows (user-level `/SC ONLOGON` task, not a true Windows Service); for non-interactive boxes (systemd / Docker / pipe) run `hope-agent server setup [--reset]` first to walk through an 11-step TTY wizard
4. `hope-agent auth codex login` is a one-shot CLI subcommand for ChatGPT OAuth login on remote hosts via SSH `-L 1455:127.0.0.1:1455` port forwarding — no desktop GUI needed

## Security

- **Unified SSRF policy**: every outbound HTTP request that consumes model / user URL input goes through `security::ssrf`, with three tiers `Strict` / `Default` / `AllowPrivate` and a `trustedHosts` allowlist. Metadata IPs (cloud-provider metadata endpoints, link-local) are rejected in every tier
- **Tool approval gating**: three states `ask` / `auto` / `full-approve`, session-persistable; supports timeout-to-deny / proceed; native approval buttons inside IM channels; optional per-channel auto-approve
- **Dangerous Mode (YOLO)**: a process-level "nuke button" that skips all tool approvals; requires an explicit "I understand the risk" checkbox in Settings to enable; a red banner stays pinned at the top of the window while active
- **API-key auth**: `Authorization: Bearer` header + `?token=` query fallback for browser WebSockets that can't set custom headers; `api_key: null` on a localhost-only bind means "allow all"
- **CSP now enforced**: the Tauri WebView CSP moved from `null` to a full policy, with all inline scripts removed
- **Automatic config snapshots**: every config write goes through `config::save_config()`, which snapshots the old file into `~/.hope-agent/backups/autosave/` (last 50 kept) — roll back any change anytime

## Known Limitations

- **iMessage channel is macOS-only** (depends on system AppleScript and the iMessage database)
- **macOS native geolocation is macOS-only** (CoreLocation delegate); other platforms fall back to IP-based geolocation for weather
- **Windows "system service" is actually a Task Scheduler user task**, not a true Windows Service (we avoid pulling in SCM dispatch code — users who need a real service can wrap the binary with nssm)
- **Windows ConPTY requires Windows 10 1809+**; older versions fall back from the PTY terminal
- **Windows installers are unsigned**, so SmartScreen may block them
- **The bundled Docker SearXNG, when searching private-network addresses** (e.g. `192.168.x.x`), will be rejected by the `Default` SSRF policy — add the host to `ssrf.trustedHosts`. The default `127.0.0.1:8080` loopback is not affected
- **Cron-driven `/recap` reports + `recap --export` CLI subcommand**: not wired up in Phase 1 — only manual / Dashboard Tab / `/recap` slash command are available
- **Multi-window / native notifications / system tray / global shortcuts / autostart**: not available in HTTP/Web mode (Tauri-only); the server mode presents these as graceful degradation
