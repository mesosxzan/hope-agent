# Hope Agent 0.1.0 — 首次公开发布

> **简体中文** · [English](./v0.1.0.en.md)

> 发布日期：2026-05-08

> 详见 [`CHANGELOG.md`](../../CHANGELOG.md#010---2026-05-08)。

同一份会话能在你的设备和聊天之间随手交接，并在日复一日的使用里自己变好——跨会话记忆持续累积、空闲时自己整理、做过的事沉淀成可复用的技能。一个原生安装包，主流大模型 GUI 模板内置齐全，填完 API Key 就能开聊；同时它也能以服务形态常驻 NAS / 自家服务器 / 云主机，在 IM 渠道里随叫随到。

## 亮点

- **三种运行模式同核**：桌面 GUI（默认）/ `hope-agent server`（HTTP/WS 守护进程，可注册为系统服务开机自启）/ `hope-agent acp`（stdio，给 IDE 当 agent 后端）
- **跨端交接**：同一份会话在桌面、浏览器、IM 之间随手交接——`/handover` 把当前桌面会话推到指定 IM 聊天，`/session <id>` 在 IM 端反向接管；桌面正在跑的对话还会**流式镜像到 IM**，模型边写边在 Telegram / 飞书 / Slack 里打字
- **服务化常驻 · 浏览器即客户端**：`hope-agent server start` 起 HTTP/WS 守护进程，二进制内嵌完整 Web GUI（rust-embed），手机 / 平板 / 浏览器打开 `http://<server>:port` 就是完整 React 界面，不用装客户端；Bearer Token + SSRF 三档策略保证公网暴露也可控
- **首次启动引导向导**：桌面 9 步 + CLI 11 步对齐——选语言、OpenClaw 一键导入、本地或远程模式、Provider、用户画像、Agent 人格预设、审批策略、技能、Server bind+API Key、IM 渠道、Summary（Web GUI URL + LAN IP + QR）。中途退出 draft 自动持久化下次续接
- **36 个 Provider 模板 · 166 个预设模型**：Anthropic / OpenAI / Codex (ChatGPT OAuth) / Gemini / OpenRouter / DeepSeek / Kimi / Qwen / 豆包 / GLM / xAI / Mistral / Ollama …一站配齐
- **本地小模型一键安装**：**不用账号、不用 API Key、不用终端**——按硬件预算挑一个能跑的 Qwen / Gemma 尺寸，一键完成 Ollama 安装、模型下载、Provider 注册与切换；同一流程也覆盖本地 Embedding；后台 watchdog 默认让模型常驻 Ollama runtime
- **多 API Key 自动轮换**：同一 Provider 下多把钥匙，遇到限流 / 额度耗尽 / 认证失效无缝切换，不够再跳下一个模型
- **12 个 IM 渠道**：Telegram / Discord / Slack / 飞书 / Google Chat / LINE / QQ Bot / Signal / iMessage / IRC / WeChat / WhatsApp。图片 / 语音 / 文件入站自动进多模态上下文，工具审批在聊天窗直接按按钮决定；回复模式三选一（split / final / preview）覆盖打字机 / 紧凑 / 渐进预览；`/reason` 可单独打开思考过程显示
- **跨会话持久记忆**：SQLite + FTS5 全文 + 向量三位一体，全局 / 项目 / Agent 三层 scope，联合预算分配不互相挤占
- **主动召回 + 反省画像 + 离线"做梦"**：每轮对话按你刚说的话主动捞相关记忆注入；反省历史对话提炼用户画像；空闲时自动整理过去 N 天最有价值的记忆并写成 markdown 日记
- **会成长的技能系统**：执行完复杂任务自动生成技能草稿等你审核；支持 `paths:` 条件激活、fork 子 Agent 执行、工具白名单隔离；兼容 agentskills.io 开放标准
- **Project 项目容器**：把相关会话归到一起，继承项目级记忆 / 指令 / 共享文件；上传的文件自动文本抽取并按需读取
- **Agent Team 多 Agent 协作**：设置里预置团队模板，模型一句话就能组建专家团协同推进
- **Plan Mode + 自然语言定时任务 + Cron 结果推送 IM**："每天早 8 点给我写日报" 到点自动跑，结果可直接发回 Telegram / 飞书；plan 期支持选取计划片段做内联评论协同修订
- **统一权限引擎 v2**：Default / Smart / YOLO 三档会话模式，Smart 用独立 LLM 判官给高置信调用自动放行；保护路径 / 危险命令 / 编辑命令三独立列表，YOLO 也强制弹保护路径审批
- **五层渐进式上下文压缩 + 崩溃恢复**：不管聊多久都不会被强切丢失，tool 调用配对永远不拆散；每个 round 增量落盘，崩溃 / kill -9 / 关 App 重启后自动从最近完成的 round 续上；配合 Prompt Caching 长会话 API 成本显著低于朴素调用
- **后台长任务**：耗时 shell / Web 搜索 / AI 画图可"丢到后台跑"，立即返回 `job_id` 不阻塞对话，完成后自动注入结果
- **内置工具箱**：可控浏览器（CDP）/ Canvas / AI 画图（7 Provider）/ Web 搜索（8 Provider failover）/ bash（可选 Docker 沙箱）/ 文件读写 / grep / MCP / URL 预览
- **Dashboard + `/recap` 深度复盘**：成本 / Token / 活跃热力图 / 健康度四维可视化；一键生成 11 章节 AI 报告导出独立 HTML
- **12 种界面语言**：简 / 繁中、英、日、韩、西、葡、俄、阿、土、越、马；深色主题与精心调校的字体排版

## 下载与安装

安装包挂在本次 Release 附件里，对应平台双击运行即可——不需要预装 Node、Rust 或 Python。

- macOS：首次打开若提示未签名，请在"系统设置 → 隐私与安全性"放行
- Windows：安装包**目前尚未做代码签名**，SmartScreen 可能拦截，选择"更多信息 → 仍要运行"；Windows 10 1809 以下系统的 ConPTY 不可用会降级
- Linux：AppImage 赋执行权限后直接运行；deb 包用 `sudo apt install ./hope-agent_*.deb`

## 首次启动

1. 桌面双击启动后会跑 9 步引导向导（语言 → OpenClaw 导入 → Provider 模板 / Codex OAuth → 用户画像 → Agent 人格预设 → 审批策略 → 技能目录 → Server bind+API Key → IM 渠道 → Summary）；中途关闭下次启动自动续接 draft，已配置过的存量用户跳过
2. 桌面应用内嵌的 HTTP 服务默认绑定 `127.0.0.1:8420` 仅本机访问；引导向导第 7 步可改 bind / 设 API Key，也可手动改 `~/.hope-agent/config.json` 的 `server.bindAddress` 为 `0.0.0.0:8420` 对外暴露（建议同时设置 `server.apiKey`）
3. `hope-agent server install` 把守护进程注册为系统服务开机自启——macOS launchd / Linux systemd user unit / Windows Task Scheduler（用户级 `/SC ONLOGON` 任务，非真正 Windows Service）；非交互场景（systemd / Docker / pipe）可先跑 `hope-agent server setup [--reset]` 走 11 步 TTY 向导
4. `hope-agent auth codex login` CLI 子命令支持远程主机走 SSH `-L 1455:127.0.0.1:1455` 端口转发完成 ChatGPT OAuth，免开桌面 GUI

## 安全

- **SSRF 统一策略**：所有面向模型 / 用户 URL 输入的出站 HTTP 走 `security::ssrf`，三档 `Strict` / `Default` / `AllowPrivate` + `trustedHosts` 白名单；元数据 IP（云厂商 metadata、link-local）所有档位都拒
- **工具审批门控**：三态 `ask` / `auto` / `full-approve`，可会话级持久化；支持超时自动 deny / proceed；IM 渠道原生按钮审批；渠道级自动批准可选
- **Dangerous Mode (YOLO)**：一键跳过所有工具审批，进程级"核弹按钮"，需要在设置里二次确认勾选"我已了解风险"才能启用；开启后标题栏常驻红条提示
- **API Key 鉴权**：`Authorization: Bearer` header + 浏览器 WebSocket 用 `?token=` query 回退，默认 `api_key: null` 本机绑定时全放行
- **CSP 首次落地**：Tauri WebView CSP 从 `null` 收紧到完整策略，移除所有 inline 脚本
- **配置自动快照**：任何配置写入都通过 `config::save_config()` 自动在 `~/.hope-agent/backups/autosave/` 存档，保留最近 50 份，可随时回滚

## 已知限制

- **iMessage 渠道仅 macOS**（依赖系统 AppleScript 和 iMessage 数据库）
- **macOS 原生定位仅 macOS**（CoreLocation delegate）；其他平台天气走 IP 地理位置降级
- **Windows "系统服务"实际是 Task Scheduler 用户任务**，不是真正 Windows Service（避免引入 SCM dispatch 代码，需要真正系统服务的用户可手动用 nssm）
- **Windows ConPTY 需要 Windows 10 1809+**；老版本下 PTY 终端降级
- **Windows 安装包未代码签名**，SmartScreen 可能拦截
- **内置 Docker SearXNG 若向私网地址搜索**（如 `192.168.x.x`），会被 `Default` SSRF 策略拒绝，需把 host 加入 `ssrf.trustedHosts`；默认 `127.0.0.1:8080` loopback 不受影响
- **cron 自动生成 `/recap` 报告 + CLI `recap --export` 子命令**：Phase 1 未接入，仅手动 / Dashboard Tab / `/recap` 斜杠命令可用
- **多窗口 / 原生通知 / 系统托盘 / 全局快捷键 / autostart**：HTTP/Web 模式下不可用（Tauri-only），服务器模式以 graceful degradation 呈现
